Todos los formulario que estan en internet tienen metodos de programación por defecto. Estos metodos nos sirven para poder hacer las acciones principales como son: Acciones para poderse conectar con los servidores, a traves de las Arquitecturas Cliente Servidor. Es que vamos a tener por defecto que todos los formularios utilizan el metodo GET, y el segundo metodo principal es POST, aunque existen otros metodos que se encargan del intercambio de información. GET: este metodo envia por medio de la URL toda la información del formulario a traves del navegador. Su funcion principal es obtener los datos del servicio; pero es el metodo mas vulnerable de todos, derivado a que se pueden generar inyecciones de codigo. POST: este metodo se encarga de registrar una nueva entrada al sistema, tambien utiliza la URL como envio por defecto; pero su facultad es poder ocultarla, esto significa que los datos no son visibles para el usuario comun en el navegador.
